# WhatsApp-бот для ремонтной компании (Solomon Partners)

Бот на FastAPI + Claude (Anthropic) + Twilio WhatsApp API.
Вся логика ответов берётся из `app/knowledge/knowledge_base.md` — это та же база знаний,
которая уже согласована. Чтобы поменять, что бот говорит клиентам — редактируйте этот файл,
код трогать не нужно.

## Что внутри

```
whatsapp_bot/
├── app/
│   ├── main.py            — вебхук, который дёргает Twilio при входящем сообщении
│   ├── claude_client.py    — вызов Claude с базой знаний и двумя инструментами (лид/эскалация)
│   ├── memory.py           — память диалога на SQLite (по номеру клиента)
│   ├── lead_logger.py      — запись лидов в data/leads.jsonl
│   ├── whatsapp.py         — отправка сообщений через Twilio
│   ├── config.py           — все настройки из переменных окружения
│   └── knowledge/
│       └── knowledge_base.md  — БАЗА ЗНАНИЙ (редактируется без правки кода)
├── requirements.txt
├── Procfile                — команда запуска для Railway/Render
├── railway.json
└── .env.example            — шаблон переменных окружения
```

## ШАГ 1 — Получить ключ Anthropic

У вас уже есть доступ к API — возьмите ключ в консоли:
https://console.anthropic.com/settings/keys

Сохраните его, понадобится на шаге 4.

---

## ШАГ 2 — Завести Twilio и подключить WhatsApp Sandbox (бесплатно для теста)

1. Зарегистрируйтесь на https://www.twilio.com/try-twilio (бесплатный trial-баланс даётся сразу).
2. В консоли Twilio откройте **Messaging → Try it out → Send a WhatsApp message**
   (или прямая ссылка: https://console.twilio.com/us1/develop/sms/try-it-out/whatsapp-learn ).
3. Twilio даст тестовый номер (обычно `+1 415 523 8886`) и код вида `join <два-слова>`.
4. С **вашего личного WhatsApp** отправьте этот код на этот номер — это активирует sandbox
   для вашего номера на 72 часа (потом нужно будет написать `join` повторно, либо подключить
   постоянный номер — см. Шаг 6).
5. В консоли Twilio найдите **Account SID** и **Auth Token** (на главной странице консоли) —
   это понадобится на Шаге 4.

> Sandbox подходит для теста бота на своём номере. Для реальных клиентов нужен постоянный
> WhatsApp-номер компании — это Шаг 6, делается после того как бот в sandbox заработал.

---

## ШАГ 3 — Завести проект на Railway (хостинг без своего сервера)

1. Зарегистрируйтесь на https://railway.app (можно через GitHub).
2. Создайте новый проект: **New Project → Deploy from GitHub repo**.
3. Залейте эту папку (`whatsapp_bot/`) в свой репозиторий на GitHub — например:
   ```
   cd whatsapp_bot
   git init
   git add .
   git commit -m "Ремонт-бот: первая версия"
   git remote add origin https://github.com/ВАШ_ЛОГИН/remont-bot.git
   git push -u origin main
   ```
4. В Railway выберите этот репозиторий — Railway сам распознает `Procfile` и `requirements.txt`
   и соберёт проект (билдер Nixpacks, уже прописан в `railway.json`).

## ШАГ 4 — Прописать переменные окружения в Railway

В дашборде проекта на Railway откройте **Variables** и добавьте (без кавычек):

| Переменная | Значение |
|---|---|
| `ANTHROPIC_API_KEY` | ваш ключ с шага 1 |
| `CLAUDE_MODEL` | `claude-sonnet-4-6` |
| `TWILIO_ACCOUNT_SID` | Account SID из Twilio |
| `TWILIO_AUTH_TOKEN` | Auth Token из Twilio |
| `TWILIO_WHATSAPP_FROM` | `whatsapp:+14155238886` (или ваш номер из шага 6) |
| `MANAGER_WHATSAPP_NUMBERS` | `whatsapp:+77011234567` (номер диспетчера, тоже должен быть в sandbox на время теста) |
| `BOT_NAME` | `Айгерим` (или другое имя) |
| `COMPANY_NAME` | `Solomon Partners` |
| `VALIDATE_TWILIO_SIGNATURE` | `true` |

Railway передеплоит проект автоматически после сохранения переменных.

## ШАГ 5 — Подключить вебхук Twilio к вашему боту

1. После деплоя Railway даст публичный URL вида `https://remont-bot.up.railway.app`.
2. В консоли Twilio откройте **Messaging → Settings → WhatsApp sandbox settings**
   (или там же, где была настройка sandbox на Шаге 2).
3. В поле **"WHEN A MESSAGE COMES IN"** укажите:
   ```
   https://remont-bot.up.railway.app/webhook/whatsapp
   ```
   Метод — `POST`. Сохраните.

Теперь все сообщения, написанные на sandbox-номер Twilio с вашего телефона, будут приходить
в бота, обрабатываться Claude и ответ прилетит обратно в WhatsApp.

---

## ШАГ 6 — Тест

Напишите с телефона, который подключён к sandbox (Шаг 2), что угодно в WhatsApp на номер
Twilio, например:
```
Здравствуйте, интересует ремонт квартиры
```
Бот должен ответить в течение нескольких секунд, начав сценарий квалификации
(раздел 8 базы знаний).

Команда `/reset` или `сброс` — обнуляет историю диалога для повторного теста сценария с начала.

## ШАГ 7 — Переход на реальный номер компании (когда тест пройден)

Sandbox ограничен 72 часами на каждого подключившегося пользователя и не подходит для
реальных клиентов. Чтобы подключить постоянный номер WhatsApp компании:

1. В Twilio: **Messaging → Senders → WhatsApp senders → Create new WhatsApp sender**.
2. Потребуется:
   - Номер телефона (можно купить прямо в Twilio, либо использовать существующий номер
     компании, если он не привязан к личному WhatsApp);
   - Facebook Business Manager аккаунт (создаётся на business.facebook.com, если ещё нет);
   - Верификация бизнеса в Meta (название компании, юридические данные — для этого подойдут
     реквизиты ТОО «Solomon Partners»);
   - Заполнение профиля WhatsApp Business (название, логотип, описание, категория).
3. Процесс верификации в Meta занимает от нескольких часов до нескольких дней.
4. После одобрения — замените `TWILIO_WHATSAPP_FROM` в переменных Railway на новый номер
   и обновите вебхук в настройках этого номера (аналогично Шагу 5, но в разделе нового sender,
   не sandbox).

Подробная официальная инструкция Twilio (на случай расхождений в интерфейсе после обновлений):
https://www.twilio.com/docs/whatsapp/self-sign-up

## Куда смотреть за лидами

Все лиды и эскалации пишутся в `data/leads.jsonl` на сервере (на Railway это эфемерное
хранилище — при редеплое файл может очиститься, поэтому **на старте это не замена CRM**,
а простой лог для проверки, что бот действительно фиксирует лидов).

Менеджер также получает мгновенное уведомление в WhatsApp при:
- успешной записи клиента на замер (raздел 10 базы — структурированные данные);
- эскалации диалога (раздел 9 базы — жалобы, юр.вопросы, агрессия и т.д.).

**Рекомендация для следующего шага** (не реализовано в этой версии, чтобы не усложнять
первый запуск): подключить Google Sheets или amoCRM/Bitrix24 через их API, чтобы
`lead_logger.py` писал лиды не в файл, а прямо в таблицу/CRM. Когда дойдёте до этого
этапа — дайте знать, дополню код.

## Как редактировать базу знаний без участия программиста

Откройте `app/knowledge/knowledge_base.md` в любом текстовом редакторе, поменяйте нужный
раздел (цены, гарантию, FAQ и т.д.), сохраните файл, закоммитьте и запушьте в GitHub:
```
git add app/knowledge/knowledge_base.md
git commit -m "Обновил цены в базе знаний"
git push
```
Railway передеплоит автоматически, бот сразу начнёт отвечать по новой версии базы.

## Известные ограничения этой первой версии

- Память диалога — SQLite-файл на диске Railway. При смене инфраструктуры/редеплое с
  очисткой volume история обнулится. Для долгосрочного хранения стоит вынести на Postgres
  (можно добавить как Railway-плагин одной кнопкой, скажите — подключу).
- Лиды пишутся в файл, не в CRM — см. рекомендацию выше.
- Нет автоматического follow-up «тёплым» лидам через 2-3 дня (упомянуто в разделе 8.5
  базы знаний как желательная фича) — это требует фоновой задачи/расписания (cron),
  можно добавить отдельно.
- Подтверждение времени замера сейчас не сверяется с реальным календарём занятости
  мастеров — бот фиксирует то время, что назвал клиент, и шлёт менеджеру для ручного
  подтверждения. Интеграция с Google Calendar — отдельная задача, есть готовый MCP-коннектор,
  можно подключить.
